Upgrade from Ansible Automation platfrom 2.4 to 2.6.

This comprehensive checklist guides you through the essential steps for a successful Ansible Automation Platform (AAP) upgrade. It covers three key phases: pre-upgrade preparation, execution, and post-upgrade verification, ensuring a smooth transition and reliable platform performance.

AAP Upgrade Checklist

  1. Before the Upgrade

    1. New Node Preparation:

      1. Confirm all new nodes have been received.

      2. Validate new nodes: Check hardware (RAM, CPU, Disk), partitions, and network settings (NFS, DNS, NTP, Ports, URLs).

      3. Verify new nodes are reachable from the installer system.

      4. Ensure the OS is consistent across all nodes.

      5. Capture and document OS hardening details.

      6. Determine if a dedicated installer user is there on the new nodes and have the sudo.

      7. Set up passwordless access to the new nodes for the installer.

    2. Backups and Documentation:

      1. Ensure the upgrade guide or document is available and reviewed.

      2. Confirm the setup bundle for your target version (e.g., 2.4/2.6) is available.

      3. Take a backup or snapshot of all existing nodes.

      4. Back up all associated databases.

      5. Capture the current load balancer (LB) configuration.

  2. After the Upgrade

    1. Verify functionality:

      1. Confirm all data and configurations are present.

      2. Ensure all jobs are executing properly.

      3. Test user login and access to confirm it’s working as expected.
