Operator Deployment LAB

We will be using Product Enablement AAP Operator Deployment lab LAB for this deployment. Please be aware that setting up a operator lab will take about fifty minutes. Following that, you can proceed with the installation procedures:

The following video will provide the step-by-step guide for installing the Ansible Automation Platform 2.6 operator:

All the followed steps in the video are shared below:

  1. Log in to the Red Hat OpenShift Container Platform.

  2. Navigate to Operators OperatorHub.

  3. Search for Ansible Automation Platform and click Install.

  4. Select a stable-2.6.

  5. Select namespace as default which will be aap.

  6. Click Install.

  7. Go to Installed Operators and there you will see Ansible Automation Platform operator installed click on it.

  8. Select project as aap if it’s not aap.

oc scale --replicas=0 deployment automation-hub-operator-controller-manager

  1. Then scroll to the option Ansible Automation Platform and click on Create instance which will deploy all the components of aap.

  2. Provide the Name = aap on which will be used to create the resources for that AAP like network routes,secrets.

    Note: There are many other options that we can customize according to the requirement but in this we will be taking care of very basic deployment.

  3. Click on Create it will deploy the resournces which will take around 20 minutes.

  4. On openshift dashboard Click on Networking and then routes and click on the ansible route or the Name given in Step 10 above.

  5. On openshift dashboard go to Workloads and then Secrets and search for admin . Click on it and copy the password.

  6. Provide the Red Hat login ID to get the subscription for attaching it to the AAP and 60-day subscription to Red Hat AnsibleÂ® Automation Platform or Red Hat Developer Subscription for Individuals subscription can be used for this lab.

  7. Submit it and run the Project sync of the Demo project to confirm that the job is getting executed.
